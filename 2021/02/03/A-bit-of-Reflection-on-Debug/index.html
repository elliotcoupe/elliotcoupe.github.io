A bit of Reflection on Debug
Elliot Coupe Lv2
 2021-02-03 15:26:42

In Software Engineering, everything working smoothly is goal to achieve for end user, and an engineer usually have to work his shit off to achieve that. And the situation engineers facing are quite the oppsite.From typing the first life of their career, there might be something wrong. Unable to debug efficient could be destroying to an engineer because it will make you surrender too quick.

Gitlab

Yesterday I did spend quite a lot of on debugging, I start a gitlab server with docker to test it’s CI/CD function. It turned to load unpleasantly slow. I checked online but find no doable solution and I didin’t realize could be related to my RAM allocation to docker util the evening.

Later I installed GitLab runner to execute the CI tasks on the same container. I spent quite a long time to find out how to register this gitlab-runner on my GitLab.

After the runner was succesfully installed, the first job was failed with not “Git command” error. I still haven’t find the solution yet since in most case the jobs was done in the image, but I learned about the before-script alone the way.

Docker

I found it quite elegant to install the databases inside docker, with this efficient. I did saved a lot time cofigure the software locally. However, I hadn’t figured out how to communition between containers if I put my application and databases in two containers.

I started with search online and found out few legacy solution “- net –link” which didn’t solve the problem I faced with. Then I finally turned the Official Docker Doc and acknowleged what should I do. I begain to believe the official document and stacklowing may not be the shortcut.

After I figured out the communication issue, I still didn’t get it work, but I think I am close. Then I narrowed the scope of this issue and found out it’s caused by I didn’t re-build this image.

Conclution

Debugging is a hard task which requires patience and resilience, but it’s also rewarding. I did enjoyed the feeling other than happy I got when I solved the issue.
