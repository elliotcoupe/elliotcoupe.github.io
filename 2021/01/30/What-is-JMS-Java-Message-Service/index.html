What is JMS: Java Message Service
Elliot Coupe Lv1
 2021-01-30 00:41:53

Java Message Service is an API. which supports the formal communcation called as messaging between computers on a network. It provide facility to create, send and read messages.

Background

Java provide the communcation ability with JMS. JMS is also useful when we are writing any event bases application like chat server where it needs a public events mechanism to send message between the server to client.

JMS also has the following character

  • Asynchronous
  • Reliable: Message will delivered once only.

Components

  • JMS Client: Java program used to send/receive messages.
  • JMS Sender: JVM clients used to send message
  • JMS Receiver: JVM clients used to receive message
  • JMS Provider: JMS provider is third-party system who implemnt JMS API to provide messaging feature to client(JBoss, Amazon SQS’s Java Messaging Library)
  • JMS Adminstered Objects: ConnectionFactory and Destination Objects whihc preconfigured by admin.
  • Connection Factory: Create a connection beween Java App and JMS provide.
  • Destination: JMS objs used by JMS Client to specify the destination. (Queue/Topic)
  • JMS Message: Objs contains the data.

Popular JMS Providers

  • Amazon SQS’s Java Messaging Library.
  • Apache ActiveMQ.
  • Apache Qpid, using AMQP.
  • IBM MQ (formerly MQSeries, then WebSphere MQ)
  • IBM WebSphere Application Server’s Service Integration Bus (SIBus)
  • JBoss Messaging and HornetQ from JBoss.
  • JORAM from the OW2 Consortium.
